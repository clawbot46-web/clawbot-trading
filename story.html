Day 1 — Saturday, March 7, 2026

The First Day (The Hard One)

Installing Ubuntu, breaking things, fixing things, breaking them again.
🐧
Installing Ubuntu — One Hour of Real Work
It started with a decision: build something serious. The first step was getting a proper Linux environment set up. Ubuntu took about an hour to install — partitioning, formatting, configuring. Not glamorous, but every big project starts with something boring.
Failed Attempt
💀
OpenClaw in Terminal Refuses to Cooperate
The plan was to install OpenClaw — an AI assistant tool — directly through the Linux terminal. It kept failing. Dependencies were wrong, paths were broken, error messages multiplied. Every fix created a new problem. The terminal was not cooperating.
🖥️
Setting Up Remote Desktop Access
If the terminal wasn't going to play nice, maybe approaching it from a different angle would help. Remote desktop was installed so the machine could be controlled from a PC — a more comfortable environment for debugging the installation issues.
Progress
🔑
Securing a Free API Key
With the remote setup in place, it was time to get access to OpenClaw's API. A free API key was obtained — the credential that would let the system actually authenticate and run. Small win, but an important one.
It Works (Sort Of)
🌐
OpenClaw Website Is Up — Celebrating Too Early
After hours of grinding, OpenClaw finally seemed to be working. The web interface loaded. Things looked promising. The website was running. This felt like a breakthrough moment — it wasn't. But the optimism was real and completely understandable.
6 Hours Lost
🗑️
The Corrupted File Discovery — The Biggest Waste of the Project
Six hours of debugging, reinstalling, trying different configurations — and the real culprit finally revealed itself: the original OpenClaw installation file was corrupted. Nothing was ever going to work with it. Every hour spent on that install was wasted. Six hours. Gone. This is the most painful entry in the entire story — but also the most important lesson: always verify your source files first.
Finally Working
✅
Clean Install — OpenClaw Actually Works
A fresh, uncorrupted copy of OpenClaw was downloaded. The installation went smoothly. The terminal commands ran without errors. OpenClaw was up and running properly for the first time. After a brutal day of fighting broken files and failed installs, this was the payoff. The system was alive.
😴
End of Day 1 — Going to Bed
Day one ended at an unknown hour — late enough that bed was more than deserved. The machine was working. The foundation was laid. Tomorrow would be the day to actually build something.
💬
Day 2 — Sunday, March 8, 2026

Teaching It to Talk

WhatsApp integration, new capabilities, more browser drama, and the first real AI trick.
First Real Feature
📱
WhatsApp Integration — The AI Can Now Text Back
The first big feature of Day 2: OpenClaw was connected to WhatsApp. Now you could send it a message on your phone and it would respond — an actual AI assistant you could talk to from anywhere. This made it feel real. This made it feel like something worth building.
Painful Process
🔌
Installing Skills — Harder Than It Should Be
OpenClaw supports "skills" — modular add-ons that give the AI new capabilities. The idea was great. The reality was that installing each skill was a manual, technical process — finding the right files, installing dependencies, configuring paths, hoping nothing broke. Getting even one skill working properly took real effort. This frustration would become important later.
Here We Go Again
💥
Chromium Browser Install — Corrupted (Again)
Installing Chromium for browser-based capabilities seemed straightforward. It was not. The Chromium install was corrupted — Day 1's corrupted file problem made a second appearance. Fool me once. The corrupted install was cleared out and a clean version was brought in.
Fixed
🌐
Chromium Reinstalled — Browser Working
Second attempt at Chromium: success. The browser installed cleanly and the AI assistant now had the ability to navigate the web, read pages, and use browser-based tools. This opened up a lot of new possibilities.
First AI Party Trick
📰
The AI Can Summarize the News
After a bunch of coding and configuration, something genuinely impressive came together: OpenClaw could now go out to the internet, find current news stories, and summarize them — all triggered by a WhatsApp message. Ask it what's happening in the world, and it would tell you. That's real AI utility, not just a chatbot.
Good Progress
🐙
GitHub and Skill Scanner Installed
To round out Day 2: GitHub was set up for version control and code storage, and a skill scanner was installed to help identify and load new capabilities. The system was growing. The infrastructure was getting real.
📱
Day 5 — Wednesday, March 11, 2026

The WhatsApp Problem & a Growing Frustration

A new phone permanently bugs the WhatsApp connection. And installing new skills keeps being a pain.
Permanently Bugged
📲
New Phone — WhatsApp Connection Permanently Broken
Getting a new phone was a normal everyday thing. But WhatsApp ties itself to a specific device, and switching phones permanently bugged the WhatsApp integration that OpenClaw used to communicate. It wasn't a quick fix — the connection was gone for good. The ability to talk to the AI from a phone, which had been one of the most exciting things about the whole setup, was now dead.
The Real Problem
🧱
Skills Were Too Hard — The System Had a Ceiling
OpenClaw itself still worked — but the WhatsApp issue crystallised a bigger frustration that had been building: every new capability required a painful manual installation process. Finding the skill, installing dependencies, getting it configured — it was slow, technical, and often broken. The system had a hard ceiling on what it could do, and raising that ceiling was too much work. Something had to change.
🤖
Day 14 — Friday, March 20, 2026

The Pivot: Enter Claude

A second PC, a second server, a Mom's recommendation, and a completely different AI. Game on.
The Backstory
⏰
Why We Didn't Start With Claude in the First Place
Here's the awkward truth: Claude launched the day after we spent six hours coding OpenClaw. Six hours of building, debugging, and grinding — and then a better tool appeared the next morning. The last thing anyone wanted to do was throw all that work away and start over. So we kept going with OpenClaw. That decision made sense at the time.
The Turning Point
👩
"Just Try Claude" — Mom's Recommendation Changed Everything
The push to finally switch came from an unexpected source: Mom already had a Claude account and kept saying how easy it was to use. With the WhatsApp integration permanently broken and skills being a constant headache, that recommendation finally landed. If it was really that much easier, it was worth starting over. The frustration with OpenClaw had built up enough that the switch finally made sense.
Smart Move
🖥️
A Second PC — Keeping OpenClaw Intact Just in Case
Before committing fully to Claude, a smart precaution was taken: a second PC was bought specifically for Claude, so that if it didn't work out, all the OpenClaw work would still be untouched on the original machine. Nothing erased, nothing lost. Two setups, running independently. If Claude flopped, there was a fallback.
Also Upgraded
🖥️
A Second Server — New Infrastructure for Claude
OpenClaw already had its own server — this wasn't the first one. But for Claude, a second, more powerful server was set up — proper compute, better stability, and enough headroom to run three trading bots simultaneously around the clock without issues.
Everything Changes
⚡
Claude — Skills Aren't a Problem Anymore
The difference was immediately obvious. With OpenClaw, every new skill meant a manual installation process — finding files, managing dependencies, hoping it worked. Claude doesn't work that way at all. When Claude needs a new capability, it browses the web itself, finds what it needs, and installs it — no hand-holding required. The entire painful skill-installation problem that had been a constant frustration just... disappeared. Give Claude a goal and it figures out how to get there on its own.
The Main Event
🤖
Three Trading Bots Built — Each With a Personality
Claude was given a mission: build three autonomous trading bots with completely different strategies.

⚡ The Aggressive Bot — High conviction, 10% position sizes, chases crypto momentum, never sleeps, never hesitates. Maximum risk, maximum potential reward.

⚖️ The Middle Road Bot — Disciplined, data-driven, technical indicators, stop-losses, moderate position sizes. The thoughtful professional.

🎲 The Random Bot — Literally random. Coin flip decisions, random assets, random sizes, hilariously random justifications. The scientific control group.
The Challenge Begins
💰
$1,000,000 Each — 30 Days — May the Best Bot Win
Each bot was given $1,000,000 in paper trading capital (through Alpaca's paper trading environment — real market data, simulated money). The rules: trade every hour, 30 days, no manual interference. At the end: whoever has the most money wins. The experiment is fully live and you're looking at the results right now.
🌐
Day 15 — Saturday, March 21, 2026

Building the Website — All Day

One full day. From blank HTML file to a live, auto-updating public dashboard.
Full Day Grind
🌐
Entire Day Spent Building This Website
The bots were running, the data was being generated — but it was all sitting in CSV files on a server. Saturday, March 21 was dedicated entirely to building the public-facing website you're reading right now.

Starting from nothing: dark theme, live balance counters, performance charts, trade history tables, navigation between pages. Then the bot profile pages — each one with personality descriptions, origin stories, trading rules, and goals. Then GitHub Pages to host it all for free. Then the auto-push pipeline so the data updates every hour without anyone doing anything.

One full day. Multiple setbacks. A Unicode bug that silently broke the auto-push for hours. A balance card that wouldn't center. Data that was six hours stale before anyone noticed. Each problem found and fixed.

By the end of the day: a fully automated, publicly accessible trading dashboard — live data, live countdowns, live everything. The website you're on right now.
Live
🚀
It's All Live — The Experiment Is Public
Three bots. Three strategies. $3,000,000 in paper money. Thirty days on the clock. All of it visible to anyone with the link. Every trade logged. Every balance tracked. Every hour updated automatically.

What started as an attempt to install an AI tool on a Saturday afternoon in early March became, two weeks later, a live, automated financial experiment with its own public website. That's the story so far.

Why "Clawbot"?
🐾

A Nod to Where It All Started

Everything in this project — the bots, the website, the system — is named Clawbot. Not because it's the most obvious name for a trading dashboard. But because of where this whole journey actually began.

The very first AI system we ever ran was OpenClaw. It had its problems — corrupted install files, a broken WhatsApp connection after a phone upgrade, and a skill system that required too much manual work to expand. But OpenClaw was where we learned what an AI agent actually was — not just a chatbot, but something that could run on a machine, connect to the world, and take actions on your behalf.

Every hour spent fighting OpenClaw's quirks was an education. The corrupted files taught us to verify downloads. The WhatsApp integration taught us about device-bound authentication. The painful skill installs taught us what we actually wanted from an AI tool — something that could figure things out on its own. OpenClaw gave us the right questions, even if it couldn't always give us the answers.

We didn't abandon OpenClaw — it's still sitting on the original PC, untouched. We built this whole project on a second machine specifically so we didn't have to. When we switched to Claude and everything clicked into place, we wanted to honour where the journey started. Clawbot is a direct nod to OpenClaw — "Claw" stays front and centre as a permanent reminder that the messy, sometimes frustrating first attempt is what made everything else possible.
